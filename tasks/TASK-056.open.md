# TASK-056: Fix iframe template EPUB 3 compliance

**Status:** not started
**Feature:** E2E testing
**Scope:** grammar-iframe, shapes-directives
**Priority:** medium
**GitHub Issue:** (create on sync)

## Description

The `b-ber-grammar-iframe` template generates HTML attributes that are not
valid in EPUB 3 XHTML, causing EPUBCheck errors when an iframe directive
appears in an epub build.

### EPUBCheck errors produced by the current template

| Error code | Attribute / cause |
| ---------- | ----------------- |
| RSC-005    | `scrolling` — not allowed in EPUB 3 XHTML |
| RSC-005    | `frameborder` — not allowed in EPUB 3 XHTML |
| RSC-005    | `webkitallowfullscreen` — proprietary, not allowed |
| RSC-005    | `mozallowfullscreen` — proprietary, not allowed |
| RSC-005    | `width="%"` — must be an integer |
| RSC-006    | Remote iframe `src` — not allowed in EPUB OCF |

### Current state (as of TASK-041)

`scrolling` and `frameborder` have been removed from `htmlIframeAttributes`
in `b-ber-shapes-directives` and from the `defaults` object in
`b-ber-grammar-iframe/src/helpers.ts` (TODO comments added pointing here).

The `webkitallowfullscreen` / `mozallowfullscreen` attributes are only
present in the installed bber v3.0.8-next.61 static template string — they
are not generated by the current monorepo source.

The `width: '100%'` default and remote-src issue still need to be resolved.

### Work required

1. **`b-ber-grammar-iframe/src/helpers.ts`**
   - Change `width` default from `'100%'` to a valid integer (e.g. `0` to
     indicate "not set", then omit the attribute if zero) — or drop `width`
     from defaults entirely and let EPUB readers render at natural size
   - Remove `width` from the defaults or validate that any user-supplied
     `width` value is a plain integer

2. **`b-ber-shapes-directives/src/index.ts`**
   - Confirm `scrolling` and `frameborder` remain removed from
     `htmlIframeAttributes` (already done)

3. **Remote `src` policy**
   - For epub builds, either: (a) omit `iframe` figure pages entirely and
     render a placeholder, or (b) document that epub builds do not support
     remote iframe sources and emit a `log.warn` instead of including a
     broken reference

4. **`b-ber-grammar-iframe/__tests__`**
   - Add tests that build against the kitchen-sink fixture `iframe` directive
     and assert no EPUBCheck errors

5. **kitchen-sink fixture**
   - Once the template is fixed, verify `bber build epub` passes EPUBCheck
     with the `ch-02-iframe-01` directive present
